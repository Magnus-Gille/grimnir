## Migration Issues Encountered & Resolved

### 1. archiver 7 -> 8 (heimdall)

**Issue:** `archiver is not a function` — v8 removed the factory function default export.

**Fix:**
```js
// Before (v7)
const archiver = require('archiver');
const archive = archiver('zip', { zlib: { level: 9 } });

// After (v8)
const { ZipArchive } = require('archiver');
const archive = new ZipArchive({ zlib: { level: 9 } });
```

**File:** `src/epub.js`

### 2. Vitest 3 -> 4: Arrow function mock constructors (skuld, ratatoskr)

**Issue:** `() => (...) is not a constructor` — Vitest 4 enforces that mock implementations called with `new` must use `function` or `class`, not arrow functions.

**Fix:**
```ts
// Before (vitest 3)
vi.mock("@anthropic-ai/sdk", () => ({
  default: vi.fn().mockImplementation(() => ({
    messages: { create: mockCreate },
  })),
}));

// After (vitest 4)
vi.mock("@anthropic-ai/sdk", () => ({
  default: vi.fn().mockImplementation(function () {
    return { messages: { create: mockCreate } };
  }),
}));
```

**Files:** `skuld/test/commitments/extractor.test.ts`, `ratatoskr/tests/concierge.test.ts`

### 3. TypeScript 5 -> 6: moduleResolution (munin-memory, hugin, ratatoskr)

**Issue:** `"moduleResolution": "Node16"` is deprecated in TS6.

**Fix:** Changed to `"NodeNext"` in `tsconfig.json` for all 3 repos. No code changes needed — functionally equivalent, just the canonical name.
